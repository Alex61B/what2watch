## Goal

Clear the two production-tree security advisories and remove version drift from the
Prisma toolchain, with **no application-code changes** and **no Prisma major-version bump**.

| ID | Finding | Advisory | Action |
|----|---------|----------|--------|

## Implementation steps (IMPLEMENT state — ⚠️ require user approval before any npm command runs)

> **Restricted actions:** `npm audit fix` and `npm install` are package mutations (CLAUDE.md "Install or update packages"). Do **not** run any of these until the user approves this plan. `--force` is forbidden.

1. **Baseline snapshot.** `git status` clean; record current versions: `npm ls next prisma @prisma/client @prisma/adapter-pg --depth=0` and `npm audit --omit=dev` (expect: 1 high `next`, 1 moderate `postcss`).
2. **Patch advisories:** `npm audit fix` (NO `--force`). This should bump `next` to the patched 15.x and the nested `postcss` to ≥8.5.10.
3. **Review the diff before accepting:** inspect `git diff package.json package-lock.json`. Confirm only `next` (+ transitive `postcss`, and at most dev-tree `ws`) moved, no major bumps, no app deps touched. If `npm audit fix` reports it *cannot* fix `next` without `--force` → **STOP and reassess** (do not force).
4. **Pin Prisma exactly:** edit `package.json` to set `"prisma": "6.19.3"`, `"@prisma/client": "6.19.3"`, `"@prisma/adapter-pg": "6.19.3"` (drop the `^`).
5. **Reconcile lockfile:** `npm install` (no args). Lockfile already holds 6.19.3, so this records the exact pins without changing installed versions.
6. **Verify** (see below).

## Rollback

If AC4 fails (build/typecheck/tests break from the `next` bump): `git checkout -- package.json package-lock.json` then `npm install` to restore. The `next` advisory has **no direct exposure here** (the app has no `middleware.ts`), so reverting is safe and non-urgent; re-plan a targeted bump if needed.

## Risks

- **Low.** `next` patch is a 15.x minor/patch; the advisory concerns middleware/segment-prefetch which this app doesn't use, so functional risk is minimal — AC4 confirms. Prisma pinning is a no-op on installed versions (lockfile already 6.19.3); it only hardens against future drift.
